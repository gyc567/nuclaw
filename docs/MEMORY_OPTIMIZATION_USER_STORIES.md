# 记忆系统优化 - 用户故事 (User Stories)

## Story 1: Search 结果去重

**作为** NuClaw 用户  
**我希望** 搜索记忆时不会返回重复的条目  
**以便** 能准确获取唯一的记忆内容

### 验收标准

- [ ] search() 不会返回重复 key 的条目
- [ ] 同一 key 只会返回最高层级(Hot > Warm > Cold)的结果
- [ ] 搜索结果按层级和相关性排序
- [ ] 100% 测试覆盖率

---

## Story 2: HotMemory 锁优化

**作为** 开发者  
**我希望** HotMemory 的读写操作减少锁竞争  
**以便** 高并发场景下性能更好

### 验收标准

- [ ] get() 操作使用读锁而非写锁
- [ ] 读写锁分离，减少争用
- [ ] 性能测试：1000次并发读操作延迟 < 100ms
- [ ] 100% 测试覆盖率

---

## Story 3: 自动维护任务

**作为** NuClaw 用户  
**我希望** 记忆数据能自动在层级间迁移  
**以便** 无需手动管理内存层级

### 验收标准

- [ ] TieredMemory 提供 start_maintenance_scheduler() 方法
- [ ] 维护任务按配置间隔运行（默认 1 小时）
- [ ] 维护任务处理：热→温，温→冷 迁移
- [ ] 支持停止维护任务
- [ ] 100% 测试覆盖率

---

## Story 4: 统一记忆系统

**作为** NuClaw 用户  
**我希望** 文件系统(MEMORY.md)与数据库记忆同步  
**以便** 记忆数据保持一致

### 验收标准

- [ ] 新的 UnifiedMemoryBridge 整合 MemoryBridge 功能
- [ ] remember_to_file() 同时写入 TieredMemory
- [ ] load_from_file() 可以回填 TieredMemory
- [ ] 两套系统数据保持一致
- [ ] 100% 测试覆盖率
