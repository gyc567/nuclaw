## 2. 缺陷复盘与修复 (Bug Fixes & Audit)

### 🐛 历史遗留问题：E2E 测试并发崩溃
在上一版的测试中，`performance_tests::test_onboard_*` 模块曾出现 11 个失败的用例。经过严苛的代码层级审计，判定这并非系统架构缺陷，而是**测试套件的并发性竞争缺陷 (Concurrency Race Condition)**。
*   **根因**：Rust 默认并行执行所有的测试用例。而这些端到端测试用例高度依赖读写同一进程的系统环境变量（如 `NUCLAW_HOME`, `ANTHROPIC_API_KEY`）。这导致线程间的环境污染，频发文件读写失败 (`No such file or directory`)。

### 🔧 修复方案与实施
*   引入了 `serial_test` 依赖库。
*   使用脚本在所有修改环境变量的测试上标记了 `#[serial_test::serial]` 宏。
*   **效果**：所有涉及到 `onboard`（启动配置向导）的端到端测试目前已经严格串行化，环境隔离完美生效，先前的 **11 个 Fail 被彻底消灭**！
